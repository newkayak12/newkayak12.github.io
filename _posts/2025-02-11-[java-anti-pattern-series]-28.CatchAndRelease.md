# Catch and release

- 예외를 받고 처리 없이 다시 던지거나 무시하는 경우
- 디버깅을 어렵게 한다.
- 근본 원인을 해결하지 않고 예외 처리 방법이 생각이 나지 않는 경우에 빠지기 쉬운 길


## 결과
1. 정보 손실
2. 잘못된 디버깅
3. 실패 위험 증가
4. 성능 저하

## 권장
1. 예외를 적절히 처리하기
2. 컨텍스트에 따라 예외 다시 던지기
3. checked exception은 신중하게
4. 강력한 오류 처리 구현
5. 예외를 철저히 기록
